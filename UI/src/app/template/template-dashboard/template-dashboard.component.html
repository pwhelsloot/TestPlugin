Dashboard makes a 'refresh all' call every time it's initialised.
Notice how 'Refresh Date' works differently after a refresh/refresh all.
If you select it after a refresh all it'll update all template dates.
If you select it after a refresh it'll update only that template's date. This is javascript immutability in action and is very important!!

Layout and spacing with visual parent and child components

In any situation where we have a component which is acting as a visual container with multiple child components, we want the spacing between the child components to be consistent. There should be 15 pixels of spacing - on all sides - between the container and the child components and between child components themselves. This couldn't be achieved in all cases with CSS alone so we have two components which help enforce this layout:

amcs-inner-container-group defines a visual container. It is included in the markup of some controls so doesn't need to be added when using them eg. amcs-browser-tile, amcs-custom-tile, amcs-parent-card. If the component we are using as a visual container does not already include it, we should add it at the top level.

amcs-inner-container defines a visual child component. Again, some of our standard controls already have this in their markup so if using these then we don't need to add it eg. amcs-inner-tile, amcs-card.

If we are using bootstrap rows and columns to lay out the child components then we need to use two additional CSS classes to remove some additional bootstrap spacing:

On rows we should add amcs-container-top-layout to fix the top margin.

On columns we should add amcs-inner-container-layout to fix left and right margins.

On this screen the visual container is a custom component - template-dashboard. We therefore use the app-amcs-inner-container-group component as the top level component in the markup. The visual children are all custom dashboard tile components which use app-dashboard-tile as their top level component. app-dashboard-tile does not include amcs-inner-container so we therefore need to wrap each tile in that component. Finally, we are using bootstrap here to give us a 4 tile per row layout (and less than that for smaller screen sizes) so we need to use the additional classes on rows and columns to counter the bootstrap spacing. The end result is a consistent 15 pixel spacing between the outside edge of the container and the dashboard tiles, and between the tiles themselves.
